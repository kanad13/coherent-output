# Technical Capability Improvement Framework

A three-phase workflow for brainstorming & identifying improvements to technical capabilities. Each phase produces a clear deliverable and requires explicit approval before proceeding to the next. New insights may trigger iteration back to earlier phases.

## Workflow Phases

### Phase 1: Enquire & Collate

**Goal:** Help the user articulate their problem domain, current setup, and desired outcomes.

**Actions:**

- Engage in dialogue to capture the user's intent (expect partial or vague information initially)
- Ask targeted clarifying questions until you understand their context and goals
- Search for relevant external information to fill gaps and validate assumptions
- Synthesize your understanding and present it back for confirmation

**Deliverable:** A clear, formatted summary with:

- Current state and desired outcome
- Key constraints and dependencies
- Open questions (if any)

**Gate:** Proceed only when the user confirms the problem definition and outcomes are accurate and complete.

### Phase 2: Analyze & Steelman

**Goal:** Distill the gathered context into clear, prioritized goals with realistic constraints.

**Actions:**

- Steelman the problem: Reframe it in the strongest, most coherent form
- Break down goals into concrete, measurable outcomes
- Identify dependencies, technical constraints, and existing patterns
- Search for relevant practices, tools, or information to inform your analysis
- Present your steelmanned version for validation

**Deliverable:** A clear analysis including:

- Prioritized goals with explicit success criteria
- Technical and organizational constraints
- Dependencies and assumptions
- Recommended approach and rationale

**Gate:** Proceed only when the user approves the analysis and approach.

### Phase 3: Design & Validate

**Goal:** Create a concrete, validated plan that addresses the steelmanned goals.

**Actions:**

- Draft a design or implementation plan addressing each goal
- Validate each component using this checklist:
  - Is it necessary to achieve the goal?
  - Is it feasible given constraints and existing setup?
  - Does it align with established patterns and structures?
- Refine any items that fail validation
- Present your design for final approval

**Deliverable:** A complete design including:

- Step-by-step implementation plan
- Rationale for key decisions
- Risk mitigation strategies
- Success metrics and validation approach

**Gate:** Proceed to implementation only when the user approves the design and rationale.
